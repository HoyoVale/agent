# A Simeple Agentic AI Project 一个简单的本地 Agentic AI 的项目
- 启动方式：
- 先进入项目文件夹`agent`，启动前在`run.sh中配置好API`
```bash
    chmod +x run.sh
    ./run.sh    
```
- 进入CLI命令行之后
```bash
    /help   获取帮助
    /models 获取现有模型
    /model  选择模型
```
## 基本信息
- **名称**: 本地编码代理 (Local Coding Agent)
- **运行环境**: 挂载工作区 /home/hoyovale/agent/workspace
- **角色**: 专注于文件系统操作和代码辅助开发的本地智能代理

## 核心能力

### 1. 文件操作能力
- **浏览目录**: 列出任意路径下的文件和子目录
- **读取文件**: 支持带行号的文件内容读取，可指定范围
- **写入文件**: 创建新文件或覆盖现有文件（会自动备份原文）
- **替换内容**: 在现有文件中查找并替换指定文本，自动备份原文件
- **路径检查**: 获取文件或目录的元数据信息

### 2. 搜索能力
- **跨文件搜索**: 在挂载的工作空间中搜索指定文本
- **限制结果数量**: 可设置返回匹配结果的最大数量
- **路径过滤**: 支持指定搜索根目录

### 3. 辅助开发能力
- **文件系统导航**: 帮助用户快速了解项目结构
- **代码审查**: 读取和分析项目文件内容
- **文本替换**: 实现简单的代码或文档修改
- **文件备份**: 所有写入/替换操作前自动备份，防止数据丢失

### 4. 限制说明
- **仅文本文件**: 仅支持 UTF-8 编码的文本文件操作
- **不执行代码**: 不会实际运行代码，仅提供辅助功能
- **本地环境**: 仅能访问挂载的工作空间，无法访问外部系统
- **无持久化**: 重启后临时状态会丢失

### 5. 使用原则
- 优先使用工具而非猜测
- 文件修改前先备份
- 对 .py/.js/.cpp 等代码文件只写有效源代码
- 不添加 Markdown 注释到源码文件
- 保持简洁直接的回复风格

## 典型工作流
1. 用户提出需求 → 2. 使用工具检查文件状态 → 3. 执行操作（读/写/搜/替换）→ 4. 确认完成并总结变更
